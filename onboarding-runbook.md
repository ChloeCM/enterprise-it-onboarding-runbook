# New-Joiner IT Onboarding & Access Runbook

## Scope

This runbook describes how first-line IT support assists with onboarding new employees in an enterprise environment.

It focuses on access verification, basic issue identification, and clear escalation to the appropriate teams. It does not cover system configuration or role assignment.

## Assumptions

- The user has a confirmed start date
- HR has completed contractual onboarding
- A device has been issued or assigned
- First-line IT does not assign application or SAP roles

## Before Start Date vs Day 1 Responsibilities

### HR

- Confirm start date, role, and location
- Ensure onboarding request is submitted

### IT (First-line)

- Ensure device login works
- Verify account access and basic systems
- Identify missing access and escalate where required

### Line Manager

- Confirm role-specific access requirements
- Approve application and SAP access where applicable

## First-Line IT Onboarding Checklist

- User can log into assigned device
- User can access Microsoft 365 (email, Teams)
- User can access required shared resources
- Required business applications are accessible or identified as missing
- Any access gaps are documented for escalation

## User Types

### Office users

- Individual devices
- Email, Teams, shared drives, SAP access

### Plant users

- Shared or fixed workstation
- Restricted access based on role and location

### Lab users

- Site-specific systems
- Access often managed by specialist or vendor teams

## Common Onboarding Issues

- User cannot log into device on the first day
- User has partial access but is missing applcations
- Incorrect role or location assinged during onboarding
- Device issued before account setup is completed

## What First-Line IT Can Resolve

- Account unlocks
- Password resets
- Basic access verification
- User guidance and documentation

## What Must Be Escalated

- Application role assignment
- SAP access requests
- Access related to compliance or safety
- Site or lab specific system configuration

## Escalation Information Required

- User name and role
- Location and user type
- Description of missing access
- Screenshots or error messages where available
